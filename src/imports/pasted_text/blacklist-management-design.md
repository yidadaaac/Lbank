请在现有后台管理系统基础上，新增一个一级菜单【黑名单管理】，并按后台可落地功能标准输出页面设计。整体视觉与现有后台保持一致。

一、菜单结构
- 新增一级菜单：黑名单管理
- 位置：左侧导航栏，与现有一级菜单同级
- 图标：使用风控/限制类通用后台图标风格
- 进入后默认落地页：黑名单列表

二、页面1：黑名单列表页
页面用途：统一查看和管理所有黑名单用户

1. 顶部区域
- 页面标题：黑名单管理
- 右上主按钮：新增黑名单

2. 筛选区
请设计标准后台筛选栏，包含以下字段：
- UID（输入框）
- 用户昵称（输入框）
- 用户实名（输入框）
- 用户类型（下拉：全部 / 商家 / 普通用户）
- 国家（下拉，可搜索）
- 添加原因（下拉：全部 / 手动添加 / 平台风控 / 业务风控）
- 禁用权限（下拉：全部 / 购买权限 / 出售权限 / 发布广告权限）
- 状态（下拉：全部 / 限制中 / 已全部解除）
- 时间筛选（添加时间）
- 按钮：查询、重置

3. 表格区
请设计后台标准表格，字段如下：
- UID
- 用户昵称
- 用户实名
- 用户类型（商家 / 普通用户）
- 国家
- 添加原因
- 禁用权限
- 解锁时间
- 添加时间
- 操作

4. 字段展示规则
- 用户类型：
  - 商家
  - 普通用户
- 添加原因：
  - 手动添加
  - 平台风控
  - 业务风控（预留）
- 禁用权限：
  - 支持一行展示多个标签，如“购买权限”“出售权限”“发布广告权限”
  - 其中：
    - 购买权限：商家 + 普通用户
    - 出售权限：商家 + 普通用户
    - 发布广告权限：仅商家
- 解锁时间：
  - 永久限制：显示“永久”
  - 非永久限制：显示具体时间，如 yyyy-mm-dd hh:mm:ss

5. 表格操作列
根据用户状态展示不同按钮：
- 权限限制
- 解除限制
- 移除黑名单

展示规则：
- 权限限制：默认可点
- 解除限制：当用户存在可解除限制时展示
- 移除黑名单：仅当用户无任何限制时展示，否则不展示按钮

三、页面2：新增黑名单弹窗
点击“新增黑名单”后打开弹窗，弹窗标题：新增黑名单

1. 弹窗字段
- 用户UID（输入框，必填）
- 用户类型（自动识别，只读展示）
- 权限限制（复选框组）

2. 用户类型识别逻辑
- 输入 UID 后，自动识别并展示用户类型：
  - 商家
  - 普通用户

3. 权限限制选项
复选框内容：
- 全部
- 限制购买
- 限制出售
- 限制发布广告

交互规则：
- 若用户为普通用户，则“限制发布广告”不可选或直接不展示
- 若勾选“全部”，则自动选中该用户类型下所有可选权限
- 需保留“确定 / 取消”按钮

4. 弹窗提示
- UID 不存在时，表单下方或 toast 提示：用户不存在，请检查 UID 后重试
- 新增成功 toast：新增黑名单成功

四、页面3：权限限制弹窗
在黑名单列表点击“权限限制”后打开弹窗，标题：权限限制

1. 弹窗内容
- 用户基础信息展示区（只读）：
  - UID
  - 用户昵称
  - 用户类型
- 权限限制（复选框组）
- 限制时间

2. 权限限制选项
- 全部
- 限制购买
- 限制出售
- 限制发布广告

规则：
- 普通用户不可选“限制发布广告”
- 选择“全部”时，自动选中全部可选权限

3. 限制时间
设计为两种模式：
- 永久（单选）
- 自定义天数（输入框，单位：天）

展示形式建议：
- 单选：永久 / 按天限制
- 若选择按天限制，则展示输入框，后缀文案“天”

4. 交互说明
- 保存后，列表中的禁用权限和解锁时间同步更新
- 永久限制时，列表解锁时间显示“永久”
- 按天限制时，系统自动计算并展示解锁时间

五、页面4：解除限制弹窗
在黑名单列表点击“解除限制”后打开弹窗，标题：解除限制

1. 弹窗内容
- 用户基础信息展示区（只读）：
  - UID
  - 用户昵称
  - 用户类型
- 当前限制项列表
- 可解除权限选择区

2. 解除限制选项
复选框组：
- 全部
- 限制购买
- 限制出售
- 限制发布广告

3. 核心规则
- 若某个限制项来源于“平台风控”，则该项不可勾选，不可解除
- 不可解除的限制项请在对应项后展示灰色说明文案：
  - 平台限制，无法解除
- 若选择“全部”，仅全选当前可解除项，不包含平台风控限制项

4. 底部操作
- 取消
- 确定解除

5. 二次确认
点击“确定解除”后，请增加二次确认弹窗：
- 标题：确认解除限制？
- 文案：解除后将恢复该用户对应权限，请确认是否继续
- 按钮：取消 / 确认解除

六、页面5：移除黑名单
此功能不单独做页面，在列表操作列体现即可

规则：
- 仅当该用户不存在任何限制项时，才展示“移除黑名单”按钮
- 若用户仍存在任意限制，则不展示该按钮
- 点击“移除黑名单”后弹出确认弹窗

确认弹窗内容：
- 标题：确认移除黑名单？
- 文案：移除后，该用户将不再出现在黑名单列表中，请确认是否继续
- 按钮：取消 / 确认移除

成功提示：
- 已移除黑名单

七、状态设计要求
请补齐以下页面状态：
1. 列表默认态
2. 无数据空状态
3. 查询结果为空状态
4. 弹窗填写态
5. 平台风控不可解除态
6. 用户为普通用户时，“发布广告权限”禁用态
7. 移除黑名单按钮不可展示态
8. 操作成功 toast 状态
9. 操作失败 toast 状态

八、文案规范
页面内统一使用以下文案：
- 新增黑名单
- 权限限制
- 解除限制
- 移除黑名单
- 手动添加
- 平台风控
- 业务风控
- 平台限制，无法解除
- 当前用户仍存在权限限制，无法移除黑名单
- 用户不存在，请检查 UID 后重试
- 新增黑名单成功
- 权限限制已生效
- 权限已解除
- 已移除黑名单

九、设计要求
- 整体为标准后台 B 端风格
- 表格信息密度适中，不要花哨
- 重点突出操作链路和状态清晰度
- 筛选区、表格区、按钮区、弹窗区层级明确
- 权限标签建议用轻量 Tag 展示
- 不同禁用权限支持多标签并列展示
- 危险操作按钮可使用弱红色或高风险样式，但整体仍保持后台统一
- 所有页面都按真实后台组件规范出图，包含 hover、disabled、empty、success、warning 等基础状态
- 输出时请给出：
  1）黑名单列表页
  2）新增黑名单弹窗
  3）权限限制弹窗
  4）解除限制弹窗
  5）移除黑名单确认弹窗
  6）关键交互说明

十、补充说明
这是在原后台基础上新增一级菜单，不是新系统。请严格按“后台功能页”设计，不要做成营销页、运营活动页或 C 端页面。重点突出表格管理、状态展示、权限操作、风控限制逻辑，确保设计稿可直接用于产品评审和研发落地。
